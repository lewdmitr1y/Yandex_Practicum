# Авторизация сотрудников партнёра по одноразовой ссылке (Magic Link)

## Цель

Спроектировать модуль авторизации сотрудников партнёров в Личном кабинете через одноразовую ссылку, отправляемую на электронную почту, с соблюдением требований безопасности, масштабируемости и удобства использования.

## Задачи

+ Проанализировать бизнес-контекст и проблему текущей авторизации
+ Сформировать уточняющие вопросы и предположения
+ Подготовить техническое задание (BR, FR, NFR, AC)
+ Спроектировать API для авторизации
+ Описать структуру базы данных
+ Построить диаграммы прецедентов и последовательностей
+ Выполнить декомпозицию задач для команды разработки

## Используемые инструменты и подходы

+ PlantUML — построение Use Case и Sequence диаграмм
+ OpenAPI 3.0 (Swagger) — проектирование API
+ Структурирование требований: BR / FR / NFR / AC
+ Декомпозиция: Epic → Feature → Task
+ Проектирование БД (users, magic_link_tokens)

## Структура проекта
